第二篇文章hexo

可以修改文章中的参数，打开scaffolds/post.md这个模板中增加类别和描述：
categories或description其他，在新建的hexo new 中建立的文章会有这几个参数，
tags和categories有多个，则用数组形式。

关于部署问题：
在部署之前要通过命令将所有文章做静态化处理，就是生产对应的html，js，css等文件，使得
所有文章是由惊天文件爱你组成的。
部署命令：
hexo generate（或简化命令：hexo g）

在本地目录生产一个public目录，里面包含所有静态文件。

生产静态文件后，发布到github，需要配置deploy指令。
在全局配置文件中deplay：

Deployment

Docs: https://github.com/Samdyddd/-lucky.github.io-.git

deploy:
type: git
repo: https://github.com/Samdyddd/-lucky.github.io-.git
branch: master

然后按照hexo-deployer-git：npm install hexo-deployer-git -S

最后利用hexo指令发布到github
hexo deploy（或是hexo d）
